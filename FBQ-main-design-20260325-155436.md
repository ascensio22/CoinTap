## Problem Statement
用户希望在 Telegram 上发布一个简单点击玩法的 Web 小游戏，核心目标是验证 Telegram 游戏的完整发布流程（技术集成 + 合规 + 用户体验）。用户背景是 Godot 2D 开发者，但选择用纯 HTML5 + JS 实现，以便理解底层原理。

## What Makes This Cool
- **最小可行游戏**：点击得金币，升级金币产量，经典 Idle Game 循环
- **完整 Telegram 集成**：不只是嵌入，而是真正利用 Telegram 的用户系统、云存储、分享能力
- **验证发布全流程**：Bot 创建 → H5 托管 → Mini App 嵌入 → Telegram 审核/合规

## Constraints
- 纯 HTML5 + JS（不用游戏引擎）
- Telegram Cloud Storage 存储用户进度（支持跨设备同步）
- 使用 Cloudflare Pages 托管（免费、快速、对中国用户友好）

## Premises
1. **Telegram Mini App 模式是最适合的嵌入方式** — 同意/不同意？
2. **Cloudflare Pages 适合作为游戏托管方案** — 同意/不同意？
3. **点击得金币的 Idle Game 玩法足够简单且有趣味性** — 同意/不同意？

## Approaches Considered

### Approach A: 最简验证方案
Summary: 单 HTML 文件 + Telegram Mini App 模式
Effort: S (1-2 小时)
Risk: Low
Pros: 代码最简单，部署最简单，最快跑通
Cons: 不符合工程实践，不易扩展

### Approach B: 稳扎稳打方案 (推荐)
Summary: 分离前端（HTML/CSS/JS）+ Telegram Bot 菜单入口 + 简单状态管理
Effort: M (3-5 小时)
Risk: Low
Pros: 更工程化，可演示更好效果，Telegram Bot 菜单按钮作为入口
Cons: 对于"验证流程"目标略微过度工程

### Approach C: 完整游戏方案
Summary: 完整 UI + 音效 + 动画 + 成就系统
Effort: L (1-2 天)
Risk: Medium
Pros: 真正可发布的游戏质量
Cons: 开发时间长，可能违背"验证流程"初心

## Recommended Approach
**Approach B: 稳扎稳打方案**

理由：
1. 代码量适中（HTML + CSS + JS 分离），结构清晰易维护
2. Telegram Bot 菜单按钮作为入口，体验更完整
3. 分离架构为未来扩展留有余地
4. Cloudflare Pages 部署简单，绑定域名后可直接用于生产

游戏核心循环：
- 点击主按钮获得金币
- 消耗金币升级"自动点击"能力（Idle 元素）
- 进度保存到 Telegram Cloud Storage

技术栈：
- 前端：HTML5 + CSS + Vanilla JS（无框架依赖）
- 托管：Cloudflare Pages（免费静态托管）
- Telegram 集成：Telegram WebApp API + Cloud Storage API

## Open Questions (已解答)

~~1. Telegram Mini App 的审核标准是什么？是否有内容限制？~~
   **已解答：** Telegram 对 Mini App 审核相对宽松，主要禁止：恶意软件、欺诈、版权内容、成人内容。游戏类内容通常可通过。审核时间 1-7 天。

~~2. Cloudflare Pages 在中国大陆的访问速度如何？~~
   **已解答：** Cloudflare 在中国访问不稳定（很多 IP 被墙）。**备选方案：** Vercel（国内访问也慢）或 Netlify。更推荐：直接用 GitHub Pages + 国内 DNS（如 DNSPod）作为主要方案，Cloudflare Pages 作为备选。

~~3. Telegram Cloud Storage 的 64KB 限制是否足够？~~
   **已解答：** 对于简单点击游戏**足够**。我们的数据结构：
   ```json
   {
     "coins": 12345,           // 金币数 (8 bytes)
     "clickPower": 1,         // 点击获得金币数 (4 bytes)
     "autoClickLevel": 5,    // 自动点击等级 (4 bytes)
     "autoClickPower": 2,    // 自动点击每秒金币 (4 bytes)
     "lastSaveTime": 1710000000  // 时间戳 (8 bytes)
   }
   ```
   估算：约 50 bytes。64KB 绰绰有余。
   **如果未来不够：** 可改用 Telegram Bot + Database（但复杂度会增加）。

## Technical Notes: Telegram Cloud Storage API

**存储读取：**
```javascript
// 通过 Telegram WebApp 初始化数据
const tg = window.Telegram.WebApp;
const initData = tg.initData;  // 包含用户信息和存储密钥
// 使用 WebApp 的 cloudStorage API
await tg.cloudStorage.getItem('gameState');
```

**存储写入：**
```javascript
await tg.cloudStorage.setItem('gameState', JSON.stringify(gameData));
```

**注意：** cloudStorage 是 Telegram Premium 用户专属功能。对于非 Premium 用户，需要使用 localStorage 作为降级方案。

3. **本地存储预览（验证游戏逻辑）**

## Game Design Spec

> **设计评审：8/10** — 经过 7 维度全面评审后的设计规范

### 核心循环
1. **点击** → 获得金币（1 coin/click 或更多）
2. **升级** → 消耗金币购买 "Auto Clicker"（每秒自动获得金币）
3. **目标** → 积累金币，解锁更高等级

### UI Layout（移动端优先）
```
┌─────────────────────────────┐
│        🪙 1,234 金币         │  ← 顶部状态栏（居中）
├─────────────────────────────┤
│                             │
│      ┌─────────────────┐    │
│      │                 │    │
│      │     CLICK       │    │  ← 主按钮（占满宽度，高度 80px+）
│      │                 │    │
│      └─────────────────┘    │
│                             │
├─────────────────────────────┤
│  ⚡ Auto Clicker Lv.3      │
│  消耗: 100 🪙 → +1/sec      │  ← 升级区域
│  ┌─────────────────────┐    │
│  │       升级          │    │  ← 升级按钮
│  └─────────────────────┘    │
└─────────────────────────────┘
```

### 移动端布局
- 单列垂直布局（不是桌面等比缩小）
- CLICK 按钮占满宽度（高度 80px+，最小触控 44x44px）
- 升级区域在 CLICK 按钮下方

### 响应式断点
- **Mobile (< 480px):** 单列垂直布局
- **Tablet/Desktop:** 可保持横向布局，但非必须

### 色彩系统

#### Light Theme
```css
--bg-primary: #FFFFFF;
--text-primary: #1a1a1a;
--accent-primary: #0088CC;   /* Telegram 蓝 */
--coin-gold: #FFD700;         /* 金币色（两主题均保持） */
--success: #4CAF50;           /* 升级成功 */
--text-secondary: #666666;
```

#### Dark Theme
```css
--bg-primary: #1E1E1E;
--text-primary: #FFFFFF;
--accent-primary: #0088CC;   /* Telegram 蓝 */
--coin-gold: #FFD700;         /* 金币色（两主题均保持） */
--success: #4CAF50;
--text-secondary: #AAAAAA;
```

### 字体
- **Font Stack:** `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif`
- 与 Telegram UI 原生字体一致，无需加载额外字体

### 交互状态

#### Loading State
- Skeleton 骨架屏（灰色块）显示主按钮区域
- 顶部金币显示 "--" 或骨架
- Telegram 品牌色 spinner (#0088CC)

#### Error State
- 图标 + 文字说明错误
- 「重新加载」按钮
- 降级提示（如 "本地存储也不可用，数据可能丢失"）

#### Empty State（新用户/零金币）
- 顶部显示 "这是你的第一个金币！" + 闪烁的 CLICK 按钮
- Onboarding tooltip: "点击按钮获得金币"
- 温暖、鼓励首次行为

#### Success State（升级成功）
- 新等级数字绿色闪烁 (#4CAF50)
- 金币扣除数字有减法动画
- 主按钮区域短暂金色粒子爆发
- 音效：升级成功音（可选）

#### Partial State（Auto Clicker 运行中）
- ⚡ 图标有脉冲动画（每1秒闪烁一次）
- 显示 "+N/sec" 计数器实时更新
- 每次 auto 金币到账，主按钮区域有微小金色闪光

#### Offline Return（离线归来）
- 弹出半透明蒙层
- 显示 "欢迎回来！" + 离线收益数字（带金色 +N 动画）
- Auto Clicker 收益单独一行显示
- 「收下！」按钮关闭弹窗

### 按钮状态机

#### 主 CLICK 按钮
- **Default:** accent-primary 填充，白字
- **按压中:** scale(0.95)，150ms ease-out
- **Haptic:** 触控设备轻微震动

#### 升级按钮
- **可购买:** accent-primary 填充，白字，hover 变亮
- **不可购买（金币不足）:** 灰色填充 (#666)，disabled
- **按压中:** scale(0.95)

### Click 反馈
- 按钮按下时 scale(0.95)，弹起时 scale(1.0)，150ms ease-out
- 点击位置产生 +N 金币飘字动画（向右上飘出，1秒后消失）

### 数字格式
- 千位分隔：1,234,567
- 万以上：1.2万
- 亿以上：1.2亿
- 金币图标 🪙 前置

### 无障碍
- 主按钮和升级按钮：44x44px 最小触控区
- 颜色对比度：文字 vs 背景 ≥ 4.5:1
- ARIA label：'点击获得金币'、'升级 Auto Clicker'

### 音效
- 默认静音
- 设置里有音效开关
- 点击音效：轻脆 "叮"
- 升级音效：成功 "哗"

### Upgrade Formula
- Auto Clicker Level N: 价格 = `floor(10 * 1.5^N)`, 每秒产量 = `N` 金币
